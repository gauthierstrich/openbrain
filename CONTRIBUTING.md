# Contributing to OpenBrain 🧠

First off, thank you for considering contributing to OpenBrain! It's people like you who make OpenBrain such a great tool for everyone.

## 🌈 Code of Conduct

Help us keep OpenBrain open and inclusive. Please read and follow our [Code of Conduct](CODE_OF_CONDUCT.md).

## 🚀 How Can I Contribute?

### Reporting Bugs
* Check the [Issues](https://github.com/gauthierstrich/OpenBrain/issues) to see if the bug has already been reported.
* If not, open a new issue with a clear title and description, including steps to reproduce the bug.

### Suggesting Enhancements
* Open an issue with a detailed description of the enhancement and why it would be useful.

### Pull Requests
1. Fork the repo.
2. Create a new branch (`git checkout -b feature/AmazingFeature`).
3. Commit your changes (`git commit -m 'Add some AmazingFeature'`).
4. Push to the branch (`git push origin feature/AmazingFeature`).
5. Open a Pull Request.

## 🎨 Style Guidelines

### Python Style Guide
We follow the [Google Python Style Guide](https://google.github.io/styleguide/pyguide.html).
* Use **4 spaces** for indentation.
* Use **docstrings** for all functions and classes.
* Ensure your code passes `flake8` or `pylint`.

### Documentation
* Update the `README.md` if your changes add new functionality.
* Use clear and concise language in your commit messages and documentation.

## 🛡️ Security

If you discover a security vulnerability, please send an email to gauthier.strich@utbm.fr instead of opening a public issue.
